The agent recovered automatically — it ran a `DISTINCT topic ILIKE '%income%'` query, discovered the actual topic name is "Median Household Income Overtime" (with "Overtime" suffix from the source), and re-queried successfully. The underlying issue is likely the empty-result handling in the Arrow→DataFrame conversion path when the WHERE filter matches zero rows. **Net result: agent's resilience to topic-name mismatch worked, but the panic is real Oxy customer-feedback (df-interchange should return an empty result, not panic on a no-match WHERE).** Worth a new limitations entry as a follow-up.
