## Decisions

- **Pivot from B (parallel-batch CLI bench) to sequential after rate-limit on initial parallel attempt.** Plan called for sequential; tried parallel under Opus's 500K cap and it worked, but kept sequential to match the plan's design and avoid burning headroom needlessly. Parallel is fine on Opus, sequential is the more durable pattern.
- **Renamed `name:` field in config.yml in lockstep with `model_ref:`.** The agent's `model:` field references the model by `name`, so flipping only `model_ref` would leave the human-readable name stale (still saying `claude-sonnet-4-6`). Both flipped together keeps the file self-consistent.
- **No prompt re-tuning.** The plan said only touch the prompt if bench reveals a real degradation. Bench showed Opus performing *better* than Sonnet (more decisive on historical-year framing, more rigorous on current_date queries, sharper limitation framing). No prompt changes needed.
- **`/api/health` shows `version: 0.5.47`, `build_timestamp: 2026-05-01`.** No Oxygen version mismatch between CLI and SPA. The model split that the wizard would have introduced (Opus 4.7 in SPA defaults vs Sonnet 4.6 in `config.yml`) is moot under `--local` mode anyway — the SPA reads `config.yml` directly.

### `oxy run` doesn't accept `-q` flag

The dev-update doc had `-q <question>` as the bench invocation form. The actual CLI is positional: `oxy run agents/answer_agent.agent.yml "<question>"`. Discovered during Session 22 pre-flight, re-confirmed during Session 26 bench. Documented for future sessions.
