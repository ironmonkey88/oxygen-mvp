## Known gotchas (real ones, hit today)

- **PR titles cannot contain semicolons.** The Bash safety hook reads `;` in `gh pr create --title "..."` as a command separator and blocks. Rephrase or use `,`. (Hit twice today.)
- **Stacked-PR base-ref doesn't auto-update on merge.** If PR B's base is PR A's branch, merging A does *not* retarget B's base to `main`. Merge the top of the stack (which carries all commits) or `gh pr edit <n> --base main` before merging dependents. (Hit during Plan A merge.)
- **EC2 dirty state on `git pull` is normal in this workflow.** This Code session repeatedly `scp`s files to EC2 mid-development for test runs; those files then look dirty when the same content lands in `main`. Discard with `git checkout --` or `rm -f` for untracked, then ff-pull. (Hit every merge today.)
- **`.mmd` files (`portal/erd-warehouse.mmd`, `portal/erd-semantic-layer.mmd`) are git-ignored.** They're build artifacts regenerated by `./run.sh` stage 9e every pipeline run. Don't try to commit them; if a local copy looks stale, regen with the script.
- **nginx `/assets` proxies to the Oxygen SPA at localhost:3000.** Static portal assets cannot live at `/assets/*` paths — they get the SPA's index.html back instead. Put static assets at top-level paths (like `/somerville-wards-background.svg`). This bit Plan 16 once; see PR #10 commit message.
- **Mermaid `useMaxWidth: true` (Mermaid 10 default for both `er` and `flowchart`) silently fights `overflow-x: auto`** by sizing the SVG to fit container. For diagrams wider than ~1400px, this makes text microscopic. The `/erd` page sets both `false` now; replicate the pattern if you add another Mermaid surface.
- **Chrome MCP can be unreachable.** Fallback is Claude Preview (`mcp__Claude_Preview__*`) — spin a local Python `http.server` against a copy of the page, point Preview at it. Worked cleanly today for the `/erd` browser-load investigation.
- **Long-running commands need `run_in_background: true`.** `./run.sh manual` takes ~14 minutes and exceeds the default 2-min Bash timeout. Background-run with notification works fine.
