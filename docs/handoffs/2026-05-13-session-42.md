## What actually shipped

Two PRs and one retraction.

**PR #3 — `0fc956e` — Session 42 first attempt (wrong hypothesis).** After operator reported the Session 41 `atob` error recurring on click-through from `/dashboards`, I reasoned the SPA must be loading other workspace files too — agents, semantic views, dbt models, schema.yml, dbt tests. Built a sweep script, ASCII-fied 22 files / 128 non-ASCII characters. Operator retested. **Same error.**

**Real diagnosis.** Curled the SPA bundle from EC2, grepped for `atob`, found:

```js
function Ip(e){ return decodeURIComponent(atob(e).split('').map(...)) }
APP: e => `${workspaceRoot}/apps/${e}`             // route definition
let n = Fp(t.path), r = APP(n)                     // callers always Fp() the path
```

`Ip` is a UTF-8-safe base64 decoder for URL path parameters. The SPA's `/apps/:pathb64` route expects a **base64-encoded path**. Callers always `Fp()` (btoa) paths before building the URL. `portal/dashboards.html`'s `Open in workspace` link was sending `rat_complaints_by_ward` — underscores aren't in the base64 alphabet, so `atob` threw.

**Hypothesis test.** Sent operator the URL `/apps/YXBwcy9yYXRfY29tcGxhaW50c19ieV93YXJkLmFwcC55bWw=` (which is `apps/rat_complaints_by_ward.app.yml` base64-encoded). Loaded cleanly with all 4 charts + 2 tables + trust signals.

- **I burned three hours on this and got the diagnosis wrong twice in a row.** The Session 41 fix happened to ship something visible (15 chars changed → operator visual-verified PR #1 merged) but the visual-verification was on a different URL path than what `/dashboards` was linking to. I read `atob` and pattern-matched to "UTF-8 somewhere" without inspecting what `Ip` actually does. The bundle is downloadable, greppable, and the function body fits on one line. Worth keeping in mind for any future SPA-side diagnostic.

- **Session 41's note file has a date off-by-one.** Frontmatter says `2026-05-14`, git commit timestamp is `2026-05-13 22:33 EDT`. Flagged inline in LOG.md but not fixed in the file itself. Trivial follow-up if you want it clean; otherwise harmless.
